# SR 11-7 Compliance

Map mltk test results to the Federal Reserve SR 11-7 Guidance on Model Risk Management sections.

**Module:** `mltk.compliance.sr_11_7`

---

## Overview

[SR 11-7](https://www.federalreserve.gov/supervisionreg/srletters/sr1107.htm) is the Federal Reserve's guidance on model risk management for banking institutions. It defines 3 pillars:

| Section | ID | Description | mltk Assertions |
|---------|----|-------------|-----------------|
| **Model Development** | `development` | Sound methodology, theory, and testing | `model.metric`, `model.regression`, `model.calibration`, `model.adversarial`, `model.bias`, `model.overfitting`, `data.schema`, `data.drift`, `data.no_nulls`, `training.*` |
| **Model Validation** | `validation` | Independent review of performance and limitations | `model.metric`, `model.slice`, `model.calibration`, `model.counterfactual`, `model.causal`, `model.attribution`, `model.conformal`, `data.synthetic` |
| **Ongoing Monitoring** | `governance` | Continuous monitoring, outcomes analysis, model inventory | `monitor.degradation`, `monitor.sla`, `monitor.streaming_drift`, `monitor.concept_drift`, `data.drift`, `data.freshness`, `inference.latency`, `inference.throughput` |

---

## Quick Start

```python
from mltk.compliance.sr_11_7 import (
    assert_sr_11_7_coverage,
    map_results_to_sections,
    find_gaps,
)

# Load results from JSON or collect from pytest
results = [
    {"name": "model.metric.accuracy", "passed": True},
    {"name": "model.slice.gender", "passed": True},
    {"name": "monitor.sla.latency", "passed": True},
]

# Check coverage
assert_sr_11_7_coverage(results, min_coverage=0.8)

# See which sections are covered
grouped = map_results_to_sections(results)
# {"development": [...], "validation": [...], "governance": [...]}

# Find gaps
gaps = find_gaps(results)
# [] -- all sections covered
```

## API Reference

### assert_sr_11_7_coverage

```python
assert_sr_11_7_coverage(results, min_coverage=0.8)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `results` | `list[dict]` | *(required)* | Test results with `name` and `passed` keys |
| `min_coverage` | `float` | `0.8` | Minimum fraction of sections covered (0-1) |

Returns `TestResult` with details: `coverage`, `covered_count`, `total`, `min_coverage`, `compliance_level`, `compliance_label`.

Raises `MltkAssertionError` when coverage is below the minimum threshold.

### map_results_to_sections

Groups results by SR 11-7 section ID (`development`, `validation`, `governance`). Results mapping to multiple sections appear in each. Unmatched results go to `uncategorised`.

### find_gaps

Returns sorted list of uncovered section IDs.

### classify_compliance

```python
classify_compliance(coverage) -> str
```

Returns one of `non_compliant`, `minimal`, `partial`, `compliant` based on the coverage ratio.

## Compliance Levels

| Level | Coverage | Description |
|-------|----------|-------------|
| Non-Compliant | < 34% | No pillars covered, practices absent |
| Minimal | 34-66% | At least one pillar covered, significant gaps |
| Partial | 67-99% | Two pillars covered, not comprehensive |
| Compliant | 100% | All three pillars covered |

---

## SR 11-7 Pillar Details

### Model Development

Covers the full development lifecycle: methodology selection, data quality checks, bias testing, overfitting detection, and training integrity (no target leakage, no train/test overlap).

### Model Validation

Independent challenge of model outputs: slice-level performance, calibration, counterfactual analysis, causal inference, feature attribution, conformal prediction, and synthetic data quality.

### Ongoing Monitoring & Governance

Post-deployment oversight: degradation detection, SLA compliance, streaming drift, concept drift, data freshness, and inference performance (latency, throughput).
